As far as I know, it isn't possible without resorting to hacks like JconServ's - which hide the input and display a regular text and button input for the user to interact with. The trouble with this approach is that it only works in IE. If the user uses another browser, they won't be able to pick a file at all.

I've come to the conclusion that the best bet is to (for the most part) leave forms with the default styling.

Comment from indiasbelly

To show a styled browse button, I've seen a lot of workarounds/hacks like the one posted here.

The problem I have with this kind of solution is that the user may decide to type the value in the textbox instead of using the browse button.  In that situation, the INPUT TYPE=file control's value property remains blank and a file is not uploaded.  You can't set the INPUT TYPE=file control's value property to the value in the textbox either, because it's read-only.

For now, I am just using the control as Microsoft intended, but if anyone has a solution, would you please share it?  I am working on pages that will be viewed on IE5.5+ only, so it doesn't have to be cross-browser.
